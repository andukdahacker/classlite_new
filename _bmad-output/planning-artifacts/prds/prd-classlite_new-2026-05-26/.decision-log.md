**Decision:** The 14 mobile mockups (s74–s87) are the responsive breakpoint designs for how desktop screens render on phone-sized viewports. They ship as part of MVP responsive behavior — not a separate mobile app.
**Source:** User (2026-05-26)

## D-7: No white-label
**Decision:** Single app for all tiers. White-label student portal removed from Studio tier.
**Source:** User (2026-05-26)

## D-8: Complete rewrite
**Decision:** v2 is a ground-up rewrite. v1 stack (React 19/Fastify/Prisma/PostgreSQL/Firebase Auth) informs but does not constrain.
**Source:** User (2026-05-26)

## D-9: Target market
**Decision:** Center owners and freelance teachers in Vietnam. Expand geographically later.
**Source:** User (2026-05-26)

## D-10: Payment provider — Polar.sh
**Decision:** Use Polar.sh as the payment provider. Sepay.vn dropped.
**Source:** User (2026-05-26)

## D-11: IELTS only for MVP
**Decision:** No subject pack abstraction. Build for IELTS only. Generalization to other subjects is a future concern — don't architect for it now.
**Source:** User (2026-05-26)

## D-12: VND pricing
**Decision:** Show pricing in VND, not USD.
**Source:** User (2026-05-26)

## D-13: AI model provider — Gemini
**Decision:** Use Google Gemini as the LLM powering AI grading suggestions and content generation.
**Source:** User (2026-05-26)
