## Story

As a **user continuing the onboarding wizard after center setup**,
I want to **pick a starter template (or opt to build from scratch), spawn my first classes with cohort names + start dates + optional teacher/student invites, and have every draft auto-saved so I can walk away mid-form**,
so that **I finish onboarding with real classes ready to teach and my persona-specific flow (Operator multi-class, Founder auto-teach-first, Solo Teacher single-class) leads me straight into the product without dead-ends**.

## Response Envelope Contract (inherited from Stories 2.1 + 2.2)

Every 2xx response from the five consumed endpoints (`GET /api/onboarding/progress`, `PUT /api/onboarding/progress`, `GET /api/templates`, `POST /api/templates/{id}/spawn`, plus optional `POST /api/templates` if custom template create lands in this story) returns `{ data, meta: { serverTime: ISO-8601 UTC } }`. **`apiFetch` drops `meta` on the floor** (`parseEnvelope<T>` at `src/lib/api-fetch.ts` returns `body.data` only — Winston-W1 verified at 2-3a). Errors return `{ error: { code, message, requestId, details? } }`; `apiFetch` throws `ApiError` — the wizard's mutation `onError` handlers branch on `ApiError.status + ApiError.code + ApiError.details + ApiError.retryAfterSeconds`. **Session hydration is complete post-2-3a** — every request from `/setup/template` onward carries a JWT with `center_id` claim; the `RequireCenterContext` middleware (Story 2.2 Task 4) rejects with `403 CENTER_REQUIRED` if not, which UI routes back to `/setup/center`.
