## Story

As a **Teacher, Admin, or Owner**,
I want **a calendar-based schedule workspace to view, create, edit, and manage sessions — including recurring series with scoped edits that never rewrite history**,
so that **I can plan and adjust the teaching schedule across day, week, and month views while the integrity of past sessions and recurring series is preserved.**

## Context: what exists vs. what this story creates

| Concern | Shipped (3.1/3.2/3.3) | This story adds |
|---|---|---|
| `sessions` table | **does not exist** (only `template_sessions`) | **NEW table** + 4-policy RLS grid + indexes + coupling CHECKs |
| Session queries/store | none | `sessions.sql`: Create(batch), ListByRange, ListByClass, GetByID(+series counts), UpdateInScope, CancelInScope, DeleteInScope — clock-guarded + optimistic |
| Session endpoints | none | `GET/POST /api/sessions`, `GET/PATCH/DELETE /api/sessions/{id}`, `POST /api/sessions/{id}/cancel` |
| Recurrence | none anywhere | materialize-on-create into N rows sharing `recurrence_group_id`; scope-driven, past-immutable, in-place mutations |
| Optimistic concurrency | none | `updated_at`-guard → 409 CONFLICT on stale scope writes |
| `/schedule` + `/my-schedule` routes | dead sidebar hrefs only | `/schedule` staff workspace (own chunk) + `/my-schedule` truthful stub |
| Calendar UI | none (unused shadcn `calendar.tsx`) | hand-rolled `ScheduleWorkspace` (Day/Week/Month + mini-month + toolbar + legend + positioned blocks + SR layer) + `SessionModal` |

| `classes.teacher_id`/`.color` | shipped (3.1) | consumed for teacher-scope + block tint (no schema change) |

## Acceptance Criteria

1. **`sessions` schema + RLS migration.** New pair `20260721120000_create_sessions` (WF-2 — never edit prior migrations). Columns:
   - `id uuid PK DEFAULT gen_random_uuid()`; `center_id uuid NOT NULL REFERENCES centers(id) ON DELETE CASCADE` (set DIRECTLY from `tc.CenterID` on insert — NOT the `template_sessions` trigger); **`class_id uuid NOT NULL REFERENCES classes(id) ON DELETE RESTRICT`** (Winston — never let a class-delete silently destroy taught-session history; classes are never hard-deleted today, so RESTRICT is a dormant belt); `topic varchar(200)`; `starts_at timestamptz NOT NULL`; `ends_at timestamptz NOT NULL`; **`recurrence_tz text NOT NULL DEFAULT 'Asia/Ho_Chi_Minh'`** (Winston — capture the AUTHORING IANA zone now; single-TZ *rendering* stays v1, but discarding the source zone is the irreversible mistake); `status text NOT NULL DEFAULT 'scheduled' CHECK (status IN ('scheduled','cancelled'))`; **`cancelled_at timestamptz`** (the queryable when-cancelled); `recurrence_group_id uuid` (NULL = one-off); `recurrence_pattern text CHECK (recurrence_pattern IS NULL OR recurrence_pattern IN ('daily','weekly','custom'))`; `created_at`/`updated_at timestamptz NOT NULL DEFAULT now()`.
   - Constraints: `sessions_time_order CHECK (ends_at > starts_at)`; **`sessions_recurrence_coupling CHECK ((recurrence_group_id IS NULL) = (recurrence_pattern IS NULL))`**; **`sessions_cancelled_coupling CHECK ((status = 'cancelled') = (cancelled_at IS NOT NULL))`**.
   - Indexes: `idx_sessions_center_id`, `idx_sessions_class_id`, partial `idx_sessions_recurrence_group_id … WHERE recurrence_group_id IS NOT NULL`, composite `idx_sessions_center_starts_at (center_id, starts_at)` (the date-range list — PERF-2).
   - **4-policy tenant RLS grid** identical to `classes` (`center_id = NULLIF(current_setting('app.current_tenant_id', true), '')::uuid`; `ENABLE` + `FORCE`; UPDATE has USING + WITH CHECK). `.down.sql` drops the table. `updated_at` has no trigger — every UPDATE query sets it explicitly. `ends_at`/duration: **duration is authored on the modal, `ends_at` derived on write; duration is a display-derived value thereafter** (document — do not add a duration column).

2. **Session CRUD API + authz + audit + optimistic concurrency.** Endpoints on `sessionChain` (= classChain shape; role enforced in-service):
   - **`GET /api/sessions?from=<date>&to=<date>&classId=<uuid?>`** → `EnvelopeSessionList`. `from`/`to` required, **half-open `[from, to)` on local calendar days**; sessions that straddle `to` are included iff `starts_at < to` (pin this — prevents "missing from the last day" bugs). **Server caps the window: `to - from > maxScheduleRangeDays (92)` → 422 `SCHEDULE_RANGE_TOO_WIDE`** (Winston — no unbounded month×JOIN). Each DTO denormalizes `className` + `classColor` (JOIN classes) + `topic`, `startsAt`, `endsAt`, `status`, `recurrenceGroupId`, `recurrencePattern`, `updatedAt` (GO-5: all present, nulls explicit).
   - **`POST /api/sessions`** → 201 `{data:{recurrenceGroupId|null, count, first:Session}}`. Body: `classId`, `topic?`, `startsAt`, `durationMinutes`, `recurrence` (AC3).
   - **`GET /api/sessions/{id}`** → detail + **`series:{groupId, total, upcoming, completed}` counts** (so the scope UI can label "All (23 · 9 completed)" without a second call — Sally/Winston).
   - **`PATCH /api/sessions/{id}`** → **field edits ONLY** (topic/startsAt/durationMinutes/classId), body carries `applyScope: 'this'|'future'|'all'` + **`expectedUpdatedAt`**; scoped, past-immutable, in-place (AC4).
   - **`POST /api/sessions/{id}/cancel`** → cancel-in-series (`status='cancelled'`, `cancelled_at=now`), body `applyScope` + `expectedUpdatedAt`; keeps rows (FR-17). **Split off PATCH** (Winston — one verb, one audit event, one authz path).
   - **`DELETE /api/sessions/{id}?scope=this|future|all`** + `expectedUpdatedAt` header → scoped hard-delete of non-completed rows only; 204.
   - **Optimistic concurrency:** every mutation re-reads the target's `updated_at` in-tx; `!= expectedUpdatedAt` → **409 `SESSION_CONFLICT`**; the whole scope op is one atomic tx (J19-004 oracle).
   - **Authz:** service `assertClassRole{owner,admin,teacher}` (DB-authoritative `tc.Role`; students → **403 `INSUFFICIENT_ROLE`**). `assertTeacherScope`: teacher may only read/write sessions of a class where `classes.teacher_id = tc.UserID`; else **404 `SESSION_NOT_FOUND`** (never 403). Owner/admin center-wide.
   - **Audit (in-tx):** `session.created` (one row per call, `changes.recurrenceGroupId`+`count`), `session.updated`, `session.cancelled`, `session.deleted` — entity `"session"`, `sessionAuditSnapshot` mirroring `classAuditSnapshot`.
   - Envelope/errors/body-cap/`DisallowUnknownFields`/`MaxBytesReader` mirror `class_handler.go`. api.yaml paths + schemas + full 401/403/404/409/413/422/429/500; then `scripts/codegen.sh` (WF-1/WF-3, last).

3. **Recurrence engine — materialize-on-create (AC5/AC6 of epic).** `POST` with `recurrence.pattern != 'none'` expands into **N concrete rows sharing one new `recurrence_group_id`** (materialized, NOT a stored RRULE) in a single tx; `recurrence_tz` stamped from the app zone:
   - `none` → 1 row, group_id/pattern NULL. `daily` → each day to `endDate`. `weekly` / `custom` → `weekdays[]` (0–6) each week to `endDate` (v1 `custom` = multi-weekday; richer rules out of scope).
   - **Bound:** recurring REQUIRES `recurrence.endDate`; generated count capped at `maxRecurrenceOccurrences (200)` → 422 `RECURRENCE_LIMIT_EXCEEDED` whose message states the max reachable end date for the chosen pattern (Winston). `endDate < startsAt` → 422.
   - Times stored `timestamptz`, generated in the single app timezone (Asia/Ho_Chi_Minh, no DST). **Editing the recurrence PATTERN or moving a session OUT of its series is OUT OF SCOPE** (create-time only — removes the re-materialization R19 leak surface; Murat).

4. **Recurrence scope on mutation — the R19 surface (past-immutable).** Editing/cancelling/deleting a session in a series presents an **"Apply to…" selector** — `this` / `future` / `all`. Scope drives the store WHERE-clause; **a `starts_at >= now()` (injected clock) floor is ANDed onto every MUTATION regardless of scope** so past/completed sessions are never rewritten, cancelled, or deleted (protects 3.5 attendance):
   - `this` → the single target row; if `target.starts_at < now()` → 422 `SESSION_ALREADY_STARTED`.
   - `future` → `recurrence_group_id = $grp AND starts_at >= $target.starts_at AND starts_at >= now()` (the **`>=` INCLUDES the clicked occurrence** — the J19-005 boundary).
   - `all` → `recurrence_group_id = $grp AND starts_at >= now()` (**all non-completed; past EXCLUDED**).
   - Non-recurring (`recurrence_group_id IS NULL`) ignores scope (only `this`).
   - **Reads/display still show past** (the calendar renders history); only mutations are floored. Every mutation is an **in-place UPDATE** over the scoped set — never delete+recreate. **R19 integrity:** each scope × field × boundary combination asserts BOTH the in-scope set changed AND every out-of-scope row (past, earlier-future, other-group) is byte-unchanged (Task 0).
