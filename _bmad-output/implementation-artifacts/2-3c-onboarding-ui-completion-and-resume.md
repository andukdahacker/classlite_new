## Story

As a **user finishing the onboarding wizard**,
I want to **see a summary of what I just set up (center name + N classes + M teachers invited) and jump into the dashboard from one CTA ‚Äî plus have the option to pause any mid-wizard step and pick up later from the dashboard**,
so that **I feel confident onboarding is genuinely done, and I'm never trapped in the wizard when I need to step away**.

## Response Envelope Contract (inherited from Stories 2.1 + 2.3b)

The only endpoint this story touches at write-time is `PUT /api/onboarding/progress` (already consumed at every wizard step). Success shape: `{ data: { currentStep, payload, updatedAt }, meta: { serverTime } }`. Error shape: `{ error: { code, message, requestId, details? } }`. `apiFetch` drops `meta` on the floor (`parseEnvelope<T>` at `src/lib/api-fetch.ts`); consumers read `data` only. **The "Save and finish later" affordance calls `autoSave.flush()` and then `navigate('/dashboard', { replace: true })`** ‚Äî it does NOT bump `currentStep` to `'done'`. That transition is reserved for the spawn/solo terminal PUT (`useAutoSave.flushWithLatch(..., { currentStep: 'done' })` per 2-3b Winston-W2 + R1-C2-P1). The completion screen is a pure consumer of `session.center` + `progress.data.payload.templateDraft`; no writes fire from `/setup/done`.

   **Stat-filter contract for teachersInvitedCount** [W-S3 + A-S2]:
   ```typescript
   const selfEmail = user?.email?.toLowerCase().trim() ?? ''
   const teachersInvitedCount = progress.data?.payload?.templateDraft?.classesDraft
     ?.filter((row) => {
       if (row.teacherEmail == null) return false
       const normalized = row.teacherEmail.toLowerCase().trim()
       if (selfEmail === '') return true  // no self to exclude ‚Äî count everything
       return normalized !== selfEmail    // case+trim self-exclusion
     })
     .length ?? 0
   ```
   Case-insensitive + trim'd self-exclusion prevents Founder Winston-W4 display injection (row 0 `teacherEmail = user.email` in draft while wire ships `null` ‚Äî see 2-3b AC7) from inflating the count. The `selfEmail === ''` fallback handles the transient state where `user` is defined but `user.email` is temporarily undefined during boot-probe ‚Äî never throws, degrades to "count everything" rather than misfiring self-exclusion. **Derivation MUST live inside the `progress.data && user` truthy branch of the page component** [A-S2] ‚Äî `<DoneHeroPanel>` is a pure display component (Task 2.2 API) and receives fully-materialized primitives, never nullable inputs.

2. **Route guard ‚Äî `/setup/done` requires spawn success; mounts an AC10-style resume-routing effect.** Single `useEffect` with a `hasRoutedOnMountRef` **route-once ref** (mirrors 2-3a PersonaSelectPage R1-P10 pattern, but with **navigate-only latch semantics ‚Äî see below**). **6-branch ladder** in order (fires only when `!inFlight && progress.isLoading === false && !progress.isError && progress.data != null`):

   - **Guard 0 (early-out)** ‚Äî `if (!progress.data) return` before any branch reads `progress.data.persona`. Prevents `Cannot read property 'persona' of undefined` when `isLoading: false && data: undefined` (cache miss on remount / query disabled). [W-B1]
   - **Guard 0b (boot-probe gate)** ‚Äî the entire effect is gated behind `!inFlight` where `inFlight = isLoading || useQueryClient().getQueryState(authKeys.session()) === undefined` (mirrors `OnboardingLayout.tsx:88-90`). Without this, a mid-probe `session === null` read is indistinguishable from "user has no center" and would bounce the user to `/setup/center` mid-probe. [W-B2]
   - **Branch 1** ‚Äî `progress.data.persona === null` ‚Üí `navigate('/welcome', { replace: true })` + set `hasRoutedOnMountRef.current = true`. Deep-link + stale-cookie edge.
   - **Branch 2** ‚Äî `session?.center == null` ‚Üí `navigate('/setup/center', { replace: true })` + set ref. Uses optional chaining because branch 0b has already gated `session` being present as `undefined`. If `session` is defined but `session.center === null`, user hasn't created a center yet.
   - **Branch 3** ‚Äî `progress.data.currentStep !== 'done'` ‚Üí dispatch by persona (mirrors PersonaSelectPage:76-89 amendment): `solo_teacher` ‚Üí `/setup/first-class`; `operator | founder` ‚Üí step-based (`center` ‚Üí `/setup/center`, `template` ‚Üí `/setup/template`, `spawn | solo_first_class` ‚Üí `/setup/spawn`). Set ref. **3 personas √ó 4 non-done in-flight steps = 12 named permutations for Task 2.4's test enumeration** [M-B2].
   - **Branch 4** ‚Äî `progress.data.payload?.templateDraft?.spawnedClassIds` empty OR undefined ‚Üí **DO NOT silent-bounce to `/dashboard`.** Render a visible fail state instead: `<Alert>` with copy `t('onboarding.done.error.setupIncomplete')` = "We couldn't confirm your setup. This is rare ‚Äî try refreshing, or [Continue to dashboard]." Two CTAs: "Try again" (calls `progress.refetch()`) and "Continue to dashboard" (`navigate('/dashboard')`). Do **NOT** set `hasRoutedOnMountRef.current = true` ‚Äî user must act. **Rationale (S-B1)**: silent bounce swallows a real state-corruption signal; user lands on an empty dashboard with no explanation and can't reconstruct what happened. Empty and undefined are **two enumerated test permutations** [M-I4] ‚Äî `spawnedClassIds: []` and `spawnedClassIds: undefined` fire the same visible fail render.
   - **Branch 5 (else)** ‚Äî **Stay + render `<DoneHeroPanel>`**. **DO NOT set `hasRoutedOnMountRef.current = true` on this branch** [W-B3]. Latch semantics = navigate-only: subsequent `progress` refetches on tab focus / auth invalidation must be able to re-check the ladder. If session ages out mid-celebration and `session.center` flips to `null`, the guard re-fires and correctly routes the user out.

   On `progress.isError` (5xx / network / retry: false) ‚Äî do NOT auto-route. See AC9 for the error state + retry contract.

   **Guard-order pinning tests** [M-I1]: (a) `persona: 'operator' && session.center: null` ‚Üí routes to `/setup/center` (branch 2 fires, not branch 1 ‚Äî persona is defined so `null` check on persona misses); (b) `persona: null && session.center: null` ‚Üí routes to `/welcome` (branch 1 fires first).
